# 深度报告设计规范

基于参考报告《针对40-65岁国央企中高层的AI工具与健康管理需求深度洞察》的风格分析。

## 视觉设计特征

### 1. 色彩方案
- **主色调**：黑白灰（#000000, #FFFFFF, #666666, #999999）
- **强调色**：深蓝色 #1e40af（用于关键数据和重要标题）
- **禁止**：大面积彩色、渐变、多色混用

### 2. 字体层级
- **报告标题**：48px, 粗体, 黑色
- **章节标题**：36px, 粗体, 黑色
- **小节标题**：24px, 粗体, 深灰色
- **正文**：16px, 常规, 深灰色
- **引用**：18px, 斜体, 中灰色
- **数据标注**：14px, 粗体, 深蓝色

### 3. 排版原则
- **对齐**：严格左对齐，标题居中
- **间距**：章节间距 60px，段落间距 24px，行高 1.8
- **宽度**：内容区最大宽度 900px，居中显示
- **分隔**：用 1px 细线分隔章节，颜色 #e5e7eb

### 4. 关键组件

#### 用户画像卡片
```
┌─────────────────────────────────┐
│ 姓名，年龄                        │
│ 身份描述                          │
│ 核心诉求（一句话）                │
└─────────────────────────────────┘
```
- 背景：白色
- 边框：1px solid #e5e7eb
- 内边距：20px
- 圆角：4px

#### 用户引用框
```
"引用内容"
—— 姓名，年龄，身份
```
- 背景：#f9fafb
- 左边框：4px solid #1e40af
- 内边距：16px 20px
- 字体：斜体

#### 关键洞察框
```
关键洞察
────────
洞察内容
```
- 背景：#f3f4f6
- 边框：2px solid #374151
- 内边距：24px
- 标题：粗体，18px

#### 风险警示
```
🚨 风险标题（最高优先级）
描述内容
```
- 使用 emoji 标记：🚨（最高）、⚠（中等）、⚡（操作性）
- 背景：#fef2f2（红色系）/ #fffbeb（黄色系）
- 边框：1px solid 对应颜色

## 内容风格特征

### 1. 标题风格
- **大标题**：隐喻性、情感化（如"银发焦虑时代的守护者"）
- **小标题**：场景化、冲突感（如"双重焦虑的叠加效应"）
- **避免**：平铺直叙的标题（如"研究背景"改为"当责任与焦虑交织"）

### 2. 叙事结构
每个核心发现的标准结构：
1. 场景引入（生动的"挣扎时刻"）
2. 用户原话引用（带姓名、年龄、身份）
3. 数据支撑（自然融入叙事）
4. 关键洞察框（突出核心发现）

### 3. 语言特点
- **专业术语**：JTBD、三明治一代、功能性健康
- **口语化表达**：与数据结合（"约40%的..."）
- **情感共鸣**：通过用户故事传递温度
- **行动导向**：每章结尾提供清晰建议

## 实施检查清单

### 生成报告时必须包含：
- [ ] 纪实文学式的章节标题（至少6个）
- [ ] 真实用户引用（至少5条，带完整归属）
- [ ] 多层次结构（每个核心发现有2-3个子节）
- [ ] 关键洞察框（每章1-2个）
- [ ] 场景化开头（每个发现）
- [ ] 数据融合（定量+定性）
- [ ] 风险识别（至少3个，带emoji）
- [ ] 人文结尾（回扣开篇隐喻）

### 视觉呈现时必须做到：
- [ ] 黑白灰主色调，单一深蓝强调
- [ ] 通过字重而非颜色建立层级
- [ ] 严格对齐，清晰分组
- [ ] 用户引用突出显示
- [ ] 关键洞察框醒目
- [ ] 风险警示明确标记
